# PR #5 レビューレポート: refactor: rename Overview document files to Guidline format

**日付:** 2026-01-09
**レビュアー:** Gemini Agent
**ステータス:** 🔴 Request Changes (修正依頼)

## サマリー
本PRは `Overview*.md` ファイルを `Guidline*.md` にリネームし、`README` や `CLAUDE.md` 内の参照を更新しています。しかし、ソースコード、テスト、およびその他のドキュメントにおいて重要な更新が漏れており、不整合や潜在的なバグ（特に `init` コマンド）を引き起こす可能性があります。

## 重大な課題 (ブロッキング)

### 1. ソースコード (`src/cospec/main.py`)
`init` コマンドが依然として旧ファイル名のファイルを生成し、それらを参照する指示を表示しています。
- **場所:** `src/cospec/main.py` (75-80行目, 106-108行目)
- **問題点:**
  - `files` 辞書のキーが `"docs/OverviewDesignThinking.md"` などのままです。
  - `console.print` メッセージが `.rules/OverviewBasicRule.md` などを参照しています。
- **要求事項:** これらを `"docs/GuidlineDesign.md"` などに更新し、出力されるパスも新しい `.rules/Guidline*` 構造に合わせて更新してください。

### 2. テスト (`tests/test_init.py`)
テストが旧ファイル名の存在を検証しています。
- **場所:** `tests/test_init.py`
- **問題点:** コードが更新された後に `assert os.path.exists("docs/OverviewDesignThinking.md")` が失敗するか、コードが更新されないままなら誤った振る舞いを検証することになります。
- **要求事項:** `GuidlineDesign.md` などをチェックするようにアサーションを更新してください。

### 3. エージェントコンテキスト (`GEMINI.md`, `QWEN.md`)
エージェントへの指示書が依然として旧ファイル名を指しています。
- **場所:** `GEMINI.md`, `QWEN.md`
- **問題点:** エージェントが存在しない（またはリネームされた） `OverviewDesignThinking.md` を探そうとし、コンテキスト取得に失敗します。
- **要求事項:** すべての参照を `Guidline*.md` に更新してください。

### 4. ドキュメント (`docs/`)
仕様書や履歴ファイル内の参照が古くなっています。
- **場所:** `docs/SPEC.md`, `docs/BLUEPRINT.md`, `docs/HISTORY_CONTEXT.md`
- **問題点:** これらのファイルが依然として `OverviewDesignThinking.md` などを参照しています。
- **要求事項:** `grep` を実行し、`docs/` ディレクトリ内の残りの参照をすべて置換してください。

## 推奨アクション
1.  `grep -r "Overview" .` を実行し、残っている参照をすべて特定する。
2.  `src/cospec/main.py` を更新し、新しいファイル名を生成するようにする。
3.  `tests/test_init.py` を更新し、新しいファイル名に対応させる。
4.  `GEMINI.md`, `QWEN.md`, および残りの `docs/*.md` ファイルを更新する。
5.  変更後に `task check` がパスすることを検証する。
